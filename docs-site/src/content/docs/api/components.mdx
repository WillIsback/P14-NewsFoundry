<Aside type="note">
Les composants de page utilisent `"use client"` et gèrent l'état local + les server actions.
Les composants de contenu sont souvent des Server Components qui reçoivent des Promises en props (React Suspense).
Les UI atoms sont des composants sans état, purement présentationnels.
</Aside>

## Composants de page

### `ChatWindow`

Interface chat principale — affiche les messages, gère la soumission via server action, scroll automatique et mises à jour optimistes.

```typescript
ChatWindow(props: {
  chatId: number;    // ID de la conversation courante
  messages: {        // messages initiaux (rendu serveur)
    id: number;
    type: string;    // "user" | "ai"
    content: string;
    timestamp: string;
  }[];
}): JSX.Element
```

```typescript
// Dans /app/chat/[id]/page.tsx (Server Component)
import ChatWindow from "@/src/components/ChatWindow";
import { fetchMessages } from "@/src/actions/chat.action";

const { data } = await fetchMessages(chatId);
const messages = data?.data ?? [];

return <ChatWindow chatId={chatId} messages={messages} />;
```

**Comportement** — binds `continueChat.bind(null, chatId)` en interne, gère `useOptimistic` pour l'affichage immédiat du message utilisateur avant la réponse du serveur.

## Composants de contenu

### `AssistantCard`

Affiche les messages du chat en fonction du variant. Trois modes : messages, welcome, pending.

```typescript
AssistantCard(props: {
  variant?: "default" | "welcome" | "pending";
  messages?: {
    id: number;
    type: string;    // "user" | "ai"
    content: string;
    timestamp: string;
  }[];
}): JSX.Element
```

```typescript
import AssistantCard from "@/src/components/AssistantCard";

// État d'accueil (page vide)
<AssistantCard variant="welcome" />

// En attente de la réponse du LLM
<AssistantCard variant="pending" />

// Affichage des messages
<AssistantCard variant="default" messages={messages} />
```

### `DisplayReviews`

Liste toutes les revues (manuelles + générées depuis chats) via React `use()` et Suspense.

```typescript
DisplayReviews(props: {
  reviewsPromise: Promise<ReviewItem[]>;
  chatReviewsPromise: Promise<ReviewItem[]>;
}): JSX.Element
```

```typescript
import DisplayReviews from "@/src/components/DisplayReviews";

// Dans un Server Component
const reviewsPromise = fetchReviews().then(r => r.data?.data ?? []);
const chatReviewsPromise = fetchChatReviews().then(r => r.data?.data ?? []);

<Suspense fallback={<Spinner />}>
  <DisplayReviews
    reviewsPromise={reviewsPromise}
    chatReviewsPromise={chatReviewsPromise}
  />
</Suspense>
```

### `ErrorBoundary`

Error boundary React (class component) — intercepte les erreurs de rendu enfant et affiche un fallback.

```typescript
class ErrorBoundary extends Component<
  { children: ReactNode; fallback?: ReactNode },
  { hasError: boolean; error: Error | null }
>
```

```typescript
import { ErrorBoundary } from "@/src/components/ErrorBoundary";

<ErrorBoundary fallback={<p>Une erreur est survenue.</p>}>
  <ChatWindow chatId={id} messages={messages} />
</ErrorBoundary>
```
